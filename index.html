프로젝트 소개

영상 제작사를 위한 인스타 포맷의 영상 포트폴리오 어플리케이션입니다.
자체 제작한 영상물 홍보 및 다른 제작사의 영상을 홍보할 수 있습니다.

개발 환경

외주 MVP 개발

개인 프로젝트

개발 기간

2023.11.13. ~ 2023.11.27.

개발 스택

Front-End : React JS, Styled Components
Back-End : Firebase
상태 관리 : Zustand
웹 배포 : Netlify
웹 앱 배포 : PWA (Progressive Web App)

개발 내용

- Firebase의 realitime database를 이용하여 올린 게시글을 실시간으로 저장 및 관리
- Zustand를 통해 상태를 관리하여 게시글 CRUD와 올린 영상을 보여주는 기능 구현
- contextAPI를 이용하여 모달을 관리해 라이브러리 영향 없이 직관적인 기능 구현
- useNavigation, useLocation과 라우터로 내비게이션바,헤더바 기능 구현 및 페이지간 값 전달하여 관리
- 재사용과 유지보수를 고려한 Styled-components 채택하여 각 컴포넌트별 스타일 관리
- 완성한 웹 페이지를 PWA builder를 통해 안드로이드 apk화하여 클라이언트에게 전달

구현 기능

피드 페이지

- 올린 게시글을 모아 유튜브 영상을 보여주는 기능
- 숏츠 형태로 카드 단위 스크롤 슬라이드 기능

검색 페이지

- 인스타그램 형태로 더미 비디오 데이터를 바탕으로 영상 보여주기 기능
- 검색 기능 및 검색어 저장

마이 페이지

- 포트폴리오에 카테고리별 영상 모아보기 기능

업로드 페이지

- 유튜브 링크를 넣어 썸네일과 제목을 불러오는 기능
- 카테고리를 선택하고 내용을 입력해 글을 업로드할 수 있는 기능

트러블 슈팅

useState 값이 느리게 들어오는 문제를 UseEffect로 해결

🔗 블로깅

- 문제 1: 다른 값 들어오는 속도보다 느린 경우
- 문제 해결: state에 0 상태를 주고 formValue 값을 검사해 복사+붙여넣기를 통해 값이 한번 바뀐 후 페이지를 이동하여 값이 전달될 수 있도록 바꾸었다.

- 문제 2: 값이 한박자 늦게 찍히는 경우
- 문제 해결: text와 visibility의 상태가 바뀔 때마다 그 값을 content에 넣어 상태와 값을 일치시켰다.

원하는 카테고리만 토글하여 보여주기

🔗 블로깅

- 하고 싶었던 것: 원하는 카테고리와 해당 아이템만 열고 닫기
- 문제 발생: 하나의 카테고리만 클릭해도 모든 카테고리가 동시에 열렸다.
- 문제 해결: 토글 핸들러에서 보여주는 아이템의 값을 스프레드 연산자를 활용해 나머지 값을 저장하고, 파라미터로 받은 값을 key로 사용하였다. 클릭시 토글 핸들러 함수를 호출하고 파라미터 값으로 Key를 전달하여 key 값이 true인 경우 해당 아이템을 보여주도록 구성했다. (블로그 코드 참고)
